# 45. 買収候補リサーチとプロダクトフィット設計

## 目的

2027年6月に10億円バイアウトを狙うため、買い手候補を広く見すぎず、現実的に欲しがる可能性がある2〜3カテゴリへ絞り込む。

ミセルは、買い手が既存事業へすぐ横展開できる形に設計する。

## 候補カテゴリの整理

### A. サイネージ/CMS/施工系

代表候補：

- CloudPoint / CloudExa系
- デジタルサイネージ施工会社
- LEDビジョン会社
- AV/弱電施工会社

欲しがる理由：

- 中小施設向けの月額商品を持てる
- 施工売り切りから運用課金へ移行できる
- 既存顧客へアップセルしやすい
- ミセルのPOPタグ/QR/広告レポートが差別化になる

求める設計：

- 端末群管理
- 死活監視
- 標準施工仕様
- 障害対応SOP
- 代理店/施工パートナー運用
- 多店舗管理
- 既存CMSとの連携余地

### B. 広告/DOOH/リテールメディア系

代表候補：

- LIVE BOARD系
- 広告代理店
- 地域広告会社
- DOOH配信会社
- リテールメディア事業者

欲しがる理由：

- ホテル、温浴、貸会議室、観光施設など新しい広告在庫を持てる
- QR計測付きの地域DOOHを作れる
- 既存広告主へ新規媒体として売れる
- 将来プログラマティックDOOHへ拡張可能

求める設計：

- 広告キャンペーン管理
- 掲載期間/時間帯指定
- 放映ログ
- QRログ
- 広告主レポート
- 媒体資料
- 掲載審査フロー
- 店舗属性データ

### C. ホテル/観光/店舗DX系

代表候補：

- ホテルDX会社
- 観光DX会社
- 店舗DX会社
- POS/予約/顧客管理会社
- 施設運営会社

欲しがる理由：

- 館内案内、多言語案内、販促、広告を統合できる
- 既存ホテル/施設顧客へのアップセルになる
- AIコンシェルジュ、予約、館内FAQとの連携余地がある
- 現場に見えるDX商材として売りやすい

求める設計：

- 多言語表示
- 館内案内テンプレート
- 施設別プレイブック
- AIレポート/AI翻訳
- 予約/FAQ/周辺案内連携
- POPタグ/売店連動

## 最終的に狙うべき2〜3カテゴリ

### 第一候補：サイネージ/CMS/施工系

理由：

- ミセルのハード/施工/端末運用が直接シナジーになる
- 既存顧客に横展開しやすい
- 買収後のPMIが比較的分かりやすい
- 10億評価の説明が「月額化できる新商品」として通しやすい

刺さるミセル設計：

- 端末群管理
- Remote CMS
- 標準施工仕様
- 運用SOP
- QAチェックリスト
- 代理店/施工パートナー網

### 第二候補：広告/DOOH/リテールメディア系

理由：

- ミセルの本質である店舗メディア化と相性が良い
- 店舗数が増えた時の評価倍率が上がりやすい
- 広告主ネットワークを持つ買い手なら成長が速い

刺さるミセル設計：

- 広告枠管理
- 放映ログ
- QR計測
- 広告主レポート
- 媒体資料
- 店舗属性DB

### 第三候補：ホテル/観光/店舗DX系

理由：

- 初期導入先が安心お宿、バリアン、パセラ、ビジョンセンターであれば強い実績になる
- OmotenasuAIやホテルAIとの連携ストーリーが作れる
- 多言語案内、館内FAQ、周辺広告と相性が良い

刺さるミセル設計：

- 業種別デモ
- 多言語/AI翻訳
- 館内案内テンプレート
- AIコンシェルジュ連携
- 売店/POPタグ連動

## 買い手が欲しがる状態

### 共通必須条件

- 50店舗以上の導入または強いパイロット実績
- MRR 500万円以上が見える
- 標準施工と運用が整備されている
- 管理画面で遠隔配信/監視できる
- 放映ログとQRログが取れる
- 顧客事例がある
- 解約率が低い

### サイネージ系向け必須

- 端末群管理
- remote command
- heartbeat
- SLA
- 施工仕様
- 端末交換手順
- セキュリティ設計

### 広告系向け必須

- 広告在庫管理
- 掲載レポート
- 店舗属性
- 料金表
- 広告審査
- 広告主事例

### ホテル/観光DX向け必須

- 多言語案内
- 館内テンプレート
- FAQ連携
- 予約/周辺案内への拡張設計
- 実施設での利用事例

## ミセルを寄せるべきプロダクト像

### 端末側

- 安定稼働
- 自動復旧
- ローカルキャッシュ
- 端末ID管理
- 監視/ログ
- セキュリティ設定

### 管理画面側

- 店舗/端末管理
- 素材管理
- playlist配信
- 広告キャンペーン管理
- レポート
- ユーザー権限

### サービス側

- 導入支援
- 月次更新
- 広告運用
- POPタグ/QR導線
- AI制作/AIレポート

## 買収候補へ見せる順番

1. 導入事例
2. MRR推移
3. 店舗数
4. 管理画面
5. 端末監視
6. 広告レポート
7. 標準施工仕様
8. 買収後の横展開シナリオ

## 追加で調査すべきこと

- CloudPoint/CloudExaの販売先とパートナー構造
- LIVE BOARDの広告商品と買収/提携傾向
- 国内サイネージ施工会社のM&A事例
- ホテルDX会社の顧客基盤
- 電子棚札会社の日本展開とAPI状況

## 結論

買い手候補は以下の3カテゴリに絞って設計する。

1. サイネージ/CMS/施工系
2. 広告/DOOH/リテールメディア系
3. ホテル/観光/店舗DX系

この3カテゴリが欲しがるように、ミセルは「安定端末運用」「広告在庫管理」「施設案内/AI拡張」の3つを同時に満たすプロダクトへ育てる。
