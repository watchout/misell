# 10. Roadmap & Backlog

## 開発フェーズ

### Phase 0：端末到着後セットアップ

目的：NiPoGiでUbuntu 3画面表示の成立を確認する。

タスク：

- Windows状態で3画面出力確認
- Ubuntu 24.04 LTSインストール
- HDMI / DP / USB-C出力確認
- xrandrで3画面横並び設定
- Chromium kiosk起動確認
- 6時間連続再生テスト

完了条件：3画面が横並びで安定表示される。

### Phase 1：ローカルプレイヤーMVP

目的：3連表示エンジンを作る。

タスク：

- Node.js + Expressのローカルサーバー
- player画面
- three-zone表示
- wide表示
- playlist.json読み込み
- durationローテーション
- start/end時間帯制御
- assets配信

完了条件：3面個別表示と3面wide表示が切り替わる。

### Phase 2：LAN管理画面

目的：開発者以外でも素材を入れ替えられるようにする。

タスク：

- 素材アップロード
- 素材一覧
- playlist編集
- 表示順変更
- 表示プレビュー
- 保存後の自動反映

完了条件：同一LAN内のPCから素材とplaylistを変更できる。

### Phase 3：自動起動・復旧

目的：サイネージ端末として最低限運用できる状態にする。

タスク：

- systemdサービス
- Chromium kiosk起動スクリプト
- xrandr設定スクリプト
- サーバー落下時の自動復旧
- 端末再起動後の自動復帰
- 簡易ログ保存

完了条件：電源投入後に人手なしで表示が復帰する。

### Phase 4：営業デモ化

目的：テスト導入先に見せられる状態にする。

タスク：

- 安心お宿向けデモ素材
- バリアン向けデモ素材
- パセラ向けデモ素材
- ビジョンセンター向けデモ素材
- 60秒紹介動画
- 管理画面デモ

完了条件：商談で使えるデモが完成する。

### Phase 5：POPタグ/QR計測

目的：ミセルを単なるサイネージから売場メディアへ広げる。

タスク：

- 商品/広告登録
- QR自動生成
- POPタグPDF出力
- 商品/広告LP生成
- QRアクセスログ
- サイネージ枠との紐付け

完了条件：サイネージ表示、POPタグ、QRログがつながる。

### Phase 6：月次レポート

目的：継続課金価値を作る。

タスク：

- 放映回数集計
- QRアクセス集計
- 広告主別レポート
- 店舗別レポート
- AI改善コメント生成

完了条件：広告主・店舗へ提出できる月次レポートが作れる。

### Phase 7：クラウド化

目的：複数店舗管理へ進む。

タスク：

- 店舗ID
- 端末ID
- クラウドplaylist同期
- ローカルキャッシュ
- 死活監視
- リモート再起動
- ユーザー権限

完了条件：複数店舗の端末をクラウドで管理できる。

### Phase 8：AI/ESL拡張

目的：差別化と高粗利アップセルを作る。

タスク：

- AIコンテンツ生成
- AIレポート
- 匿名人数カウント
- ESLスターターキットAPI調査
- 電子棚札連携
- 天気/時間帯連動
- POS/予約連携のPoC

完了条件：AI拡張プランとして提案できる。
