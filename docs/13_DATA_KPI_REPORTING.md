# 13. Data, KPI & Reporting

## データの目的

ミセルの事業価値は、導入台数だけではなく、継続課金、広告稼働、放映ログ、QR反応、レポート提出によって高まる。

最初からデータを残す。

## MVPで取るデータ

### 放映ログ

- timestamp
- store_id
- device_id
- playlist_item_id
- layout
- asset_id
- duration
- result

### QRログ

- timestamp
- store_id
- campaign_id
- qr_id
- landing_page
- user_agent概要

### 運用ログ

- 素材アップロード日時
- playlist更新日時
- 表示エラー
- 端末起動/停止
- 手動再起動

## 後続で取るデータ

- 広告主ID
- 広告キャンペーンID
- 表示回数
- 時間帯別表示回数
- QR読み取り率
- POPタグ別反応
- 店舗別反応
- 月次改善メモ
- AI提案履歴

## 重要KPI

### 事業KPI

- 導入店舗数
- MRR
- ARR
- 平均月額売上
- 1店舗粗利
- 解約率
- 契約期間

### 広告KPI

- 広告掲載店舗数
- 広告掲載率
- 広告主数
- 広告継続率
- 広告売上
- 広告運用粗利

### 運用KPI

- 稼働率
- 障害件数
- 平均復旧時間
- 素材更新件数
- 月次レポート提出率
- 1店舗あたり運用時間

### クリエイティブKPI

- 表示回数
- QR読み取り数
- QR読み取り率
- 時間帯別反応
- POPタグ別反応
- LPアクセス数

## 月次レポート構成

### 店舗向け

1. 今月の放映概要
2. 人気コンテンツ
3. QR反応
4. 時間帯別傾向
5. 次月の改善提案
6. 広告枠の状況

### 広告主向け

1. 掲載期間
2. 放映回数
3. QR読み取り数
4. 表示時間帯
5. POPタグ反応
6. 改善提案
7. 次月継続提案

## AIレポート

AIで自動生成する文章：

- 今月の総括
- 反応が良かった理由の仮説
- 次月の改善案
- 広告主向けコメント
- 店舗担当者向け提案

注意：AIの推測は断定せず、ログに基づく仮説として書く。

## バイアウト評価に効くデータ

買い手に見せるべきデータ：

- 店舗別MRR
- 契約期間
- 解約率
- 広告掲載率
- 広告主数
- 月次レポート提出履歴
- 稼働率
- 運用工数
- 粗利率
- 導入業種別の反応

## 初期実装

最初はDBを使わず、JSONL/CSVでよい。

ただし、将来のクラウド化に備えて、store_id、device_id、campaign_id、asset_idを付与できる設計にする。

## 重要な考え方

広告主は「何回流したか」だけでは継続しない。

必要なのは、

- どの店舗で
- いつ
- どの素材が
- 何回流れ
- QR/POPタグでどれだけ反応し
- 次に何を改善するか

を報告すること。

ミセルの月額価値は、このレポートで高まる。
