# 36. QA・実機テストチェックリスト

## 目的

ミセル端末を現場導入する前に、表示、通信、再起動、素材更新、ログ、QRを検証する。

## 端末初期テスト

- BIOSに入れる
- USBブートできる
- Ubuntuが起動する
- 有線LANが認識する
- Wi-Fiが認識する
- SSH接続できる
- Tailscale接続できる
- RustDeskで画面確認できる

## 3画面テスト

- HDMI出力確認
- DisplayPort出力確認
- USB-C出力確認
- 3画面同時表示
- 画面順固定
- 解像度1920 x 1080 x 3
- 5760 x 1080表示
- 再起動後の画面順確認
- モニター電源OFF/ON後の復帰

## プレイヤーテスト

- player画面起動
- three-zone表示
- wide表示
- 画像表示
- 動画表示
- duration切替
- start/end時間帯制御
- 無効化アイテムのスキップ
- 素材がない場合のエラー表示

## 管理画面テスト

- LAN内PCからアクセス
- 素材アップロード
- 素材削除
- playlist編集
- 保存
- プレイヤー反映
- 不正なファイル形式の拒否
- 大きすぎるファイルの扱い

## 自動起動テスト

- 電源投入後にUbuntu起動
- Nodeサーバー自動起動
- Chromium kiosk自動起動
- xrandr設定反映
- プレイヤー表示
- 停電復帰想定テスト

## ローカル継続再生テスト

- LAN切断
- インターネット切断
- 既存素材の継続再生
- QRリンク切れ時の表示確認

## 連続再生テスト

- 6時間
- 12時間
- 24時間

記録する項目：

- CPU使用率
- メモリ使用率
- 温度
- ブラウザクラッシュ
- 動画カクつき
- 音声有無
- ログ肥大化

## QR/POPタグテスト

- QR生成
- スマホ読み取り
- LP表示
- アクセスログ保存
- 広告ID/QR ID紐付け
- レポート反映

## 障害テスト

- 動画ファイル破損
- playlist不正
- 素材パス不正
- モニター1枚未接続
- ネット切断
- Nodeサーバー停止
- Chromium停止

## 導入前合格基準

- 3画面表示が安定している
- 6時間以上停止しない
- 再起動後に自動復帰する
- LAN管理画面で素材更新できる
- QR計測が動く
- 表示ログが残る

## 注意

現場導入前に必ずテストログを残す。

障害が出た場合は、原因、対応、再発防止策をGitHub Issueに記録する。
