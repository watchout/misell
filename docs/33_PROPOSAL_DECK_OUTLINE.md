# 33. 提案書・営業デック構成

## 目的

ミセルの商談で使う提案書の標準構成を定義する。

提案書は、機材説明ではなく「導入後に施設がどう変わるか」を伝える。

初回商談でそのままスライド化できるテスト導入向け提案デック本文は `docs/60_TEST_INTRO_PROPOSAL_DECK.md` を使う。この文書は提案書全体の設計指針として残す。

## 提案書の基本構成

### 1. 表紙

- 連結サイネージ「ミセル -misell-」
- つないで魅せる。売場をメディアに。
- 提案先施設名
- 提案日

### 2. 課題提起

施設で起きている課題を整理する。

例：

- 案内が伝わらずスタッフ説明が増える
- 紙POPやポスターの更新が大変
- 施設内の空きスペースが収益化されていない
- 館内サービスが知られていない
- 広告やキャンペーンの効果が見えない

### 3. ミセルの解決策

3連サイネージ、POPタグ、QR計測、月次レポートで解決する。

- 3面で目を引く
- 時間帯別に表示を変える
- QRで行動につなげる
- 月次レポートで改善する
- 地域広告枠として収益化する

### 4. 導入イメージ

設置候補場所の写真に、3連サイネージ合成イメージを載せる。

必要なビジュアル：

- 設置前
- 設置後イメージ
- 3面表示例
- POPタグ例
- 管理画面例

### 5. 表示パターン

提案先に合う3〜5パターンを見せる。

例：

- 館内案内型
- 中央広告 + QR型
- 3面wide型
- タイムテーブル型
- POPタグ連動型

### 6. 運用フロー

1. 月次で表示内容を決める
2. 素材を登録する
3. スケジュール配信する
4. QR/放映ログを計測する
5. 月次レポートを提出する
6. 次月改善する

### 7. テスト導入プラン

- 期間
- 設置場所
- 表示内容
- 計測項目
- 成果物
- 費用
- 本契約への移行条件

### 8. 本導入プラン

Lite / Standard / Media / AI Edge の比較。

表示する項目：

- 初期費用
- 月額費用
- 更新回数
- レポート
- 広告運用
- POPタグ
- AI機能

### 9. 導入スケジュール

- 現調
- 素材準備
- キッティング
- 設置
- テスト再生
- 運用開始
- 初回レポート

### 10. 次アクション

- 設置候補場所の確認
- テスト導入可否の判断
- 素材提供
- 契約条件確認
- 初回デモ日程

## 提案書で避けること

- 機材スペックから入る
- AIを過剰に打ち出す
- 広告収益を保証する
- 個別カスタムを広げすぎる
- 価格を安く見せすぎる

## 提案書で強調すること

- 施設の見栄えが上がる
- スタッフの案内負荷が下がる
- 館内サービスが伝わる
- QRで行動が測れる
- 地域広告へ発展できる
- 月次運用を任せられる

## スライド枚数目安

初回提案：10〜12枚

詳細提案：20〜30枚

テスト導入報告：8〜12枚

## 施設別に差し替える箇所

- 表紙施設名
- 課題
- 設置場所
- 表示パターン
- KPI
- テスト導入条件
- 広告主候補

## 最終ゴール

提案書を読んだ担当者が、次の一言を言う状態にする。

「これならうちの施設でも試せそうですね。」
