# 30. 初動30日実行計画

## 目的

NiPoGi端末到着後、30日以内に「動くMVP」「見せられるデモ」「提案できる営業資料」「テスト導入候補への打診」まで進める。

## 30日後の到達状態

- Ubuntu上で3連サイネージMVPが動作
- three-zone / wide の2モードが動作
- LAN管理画面で素材入れ替え可能
- 安心お宿/バリアン/パセラ/ビジョンセンター向けデモ素材がある
- 1枚資料とLPたたき台がある
- テスト導入の提案ができる
- GitHub Issuesに開発タスクが分解されている

## Week 1：端末セットアップとMVP着手

### 目標

3画面表示をUbuntuで成立させ、Codexでmisell-playerの初期実装を開始する。

### タスク

- NiPoGi開封
- Windows状態で3画面出力確認
- Ubuntu 24.04 LTSインストール
- xrandrで3画面横並び確認
- Node.js/Chromium導入
- CodexへMVPプロンプト投入
- player.html / playlist.json / server.js 初期実装

### 完了条件

- 3画面にChromiumが表示される
- 5760 x 1080の表示が成立する
- 画像/動画が手動で表示できる

## Week 2：LAN管理と自動起動

### 目標

現場で使える最低限の運用機能を入れる。

### タスク

- admin画面実装
- 素材アップロード
- playlist編集
- durationローテーション
- start/end時間帯制御
- systemd自動起動
- Chromium kiosk起動
- 6時間連続再生

### 完了条件

- LAN内PCから素材入れ替え可能
- 再起動後に自動復帰
- 6時間止まらない

## Week 3：デモ素材と営業資料

### 目標

テスト導入先に見せられる状態にする。

### タスク

- 安心お宿向けデモ作成
- バリアン向けデモ作成
- パセラ向けデモ作成
- ビジョンセンター向けデモ作成
- 60秒紹介動画
- 1枚資料
- LPたたき台
- 価格表たたき台
- テスト導入条件整理

### 完了条件

- 4業種向けに説明できるデモがある
- 初回商談資料がある

## Week 4：テスト導入打診と改善

### 目標

テスト導入1件を決めるために動く。

### タスク

- 既存接点へ打診
- デモ実施
- 設置候補場所ヒアリング
- テスト導入条件提示
- フィードバック反映
- POPタグ/QR計測の簡易実装
- 月次レポート雛形作成

### 完了条件

- テスト導入候補1件以上
- 有償PoC候補1件以上
- 次月の営業タスクが明確

## 毎日やること

- GitHub Issue更新
- 進捗メモ
- 実機テストログ
- 動画/写真の保存
- 気づいた営業トークの記録

## 30日間の最重要KPI

- 3画面安定表示：達成/未達
- 連続再生時間：6時間以上
- デモ実施数：3件以上
- テスト導入候補：1件以上
- 営業資料：1式完成
- 事例化許諾候補：1件以上

## 30日以内にやらないこと

- 完成版クラウドCMS
- 高度なAIカメラ
- POS連携
- 電子棚札API本実装
- 複雑なユーザー権限
- 決済機能

## 判断基準

30日後に「売れるかどうか」の一次判断をする。

判断材料：

- デモを見た反応
- テスト導入の意向
- 月額への納得感
- 設置場所の現実性
- 運用負荷
- 3面演出の訴求力
