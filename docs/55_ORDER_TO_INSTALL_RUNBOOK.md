# 55. 受注から設置・運用開始までの標準フロー

## 目的

Misellを商品として販売した後、受注、現調、契約、キッティング、設置、検収、月次運用までを同じ流れで進めるためのrunbook。

## ステータス

- Lead: 初回相談
- Qualified: 設置可能性と予算感を確認済み
- Proposal: 提案書・概算見積を提示済み
- Site Survey: 現調完了
- Contracting: 契約・申込書確認中
- Kitting: 端末準備中
- Install Scheduled: 設置日確定
- Installed: 設置完了
- Burn-in: 連続稼働確認中
- Live: 運用開始
- Reported: 初回レポート提出済み
- Paid Expansion: 本契約または追加展開提案中

## 1. 商談受付

記録する項目:

- 会社名/施設名
- 担当者
- 業態
- 店舗数
- 設置候補場所
- 現在の販促/案内方法
- サイネージ導入経験
- 予算感
- 希望開始時期
- テスト導入可否
- 事例掲載可否

完了条件:

- 提案先の課題が1つ以上明確
- 設置候補場所がある
- 次回アクションが決まっている

## 2. 初回提案

提示するもの:

- 1枚営業資料
- 導入イメージ
- Lite/Standard/Mediaの概要
- テスト導入条件
- 概算費用レンジ
- 導入までの流れ

完了条件:

- 現調または詳細提案へ進む合意
- 価格帯に大きなズレがない
- 意思決定者または現場責任者が特定できている

## 3. 現調

確認するもの:

- 設置面
- 電源
- LAN/Wi-Fi
- 視認距離
- 画面サイズ
- 配線経路
- 施工可能時間
- 店舗ネットワーク制約
- 写真/動画撮影可否
- QR誘導先

残す証跡:

- 設置候補写真
- 電源位置写真
- ネットワーク条件メモ
- 施工上の注意
- 標準外作業の有無

## 4. 見積・契約

決めること:

- 導入形態: テスト / 有償PoC / 本契約
- 初期費用
- 月額費用
- 契約期間
- 支払条件
- 標準施工範囲
- 標準外費用
- 素材制作範囲
- 保守範囲
- 事例掲載可否
- 撤去条件

完了条件:

- 申込書または簡易合意書が締結済み
- 設置日候補が決まっている
- 初期素材の提供依頼が済んでいる

## 5. 端末キッティング

作業:

- Ubuntuインストール
- `apps/player` セットアップ
- `npm install --omit=dev`
- `scripts/enroll-device.sh --apply` でenv生成
- playlist初期設定
- 素材配置
- 3画面出力確認
- systemd自動起動設定
- security baseline確認
- burn-in短時間確認

完了条件:

- `npm run validate:playlist` が通る
- `/admin` が認証で守られている
- `/player?preview=1` が表示される
- device_id/store_id/screen_group_id がログへ出る
- 端末台帳に記録済み

## 6. 設置当日

作業:

- 画面固定
- 端末固定
- ケーブル固定
- 電源確認
- LAN/Wi-Fi確認
- 3画面順確認
- kiosk起動確認
- playlist表示確認
- QR読み取り確認
- 再起動復旧確認
- 管理画面アクセス確認
- 担当者へ最低限の説明

残す証跡:

- 設置写真
- 3画面表示動画
- `scripts/collect-device-evidence.sh` の出力
- systemd status
- burn-in開始ログ

## 7. Burn-in・検収

最低ライン:

- 6時間連続再生
- 表示停止なし
- 再起動後復帰
- playlist保存/反映
- 認証/アップロード検証

推奨:

- 本契約前は24時間連続再生
- 店舗営業時間中の視認確認
- 担当者フィードバック取得

検収条件:

- 顧客が表示内容を確認した
- 現地写真/動画が残っている
- 重大な表示崩れがない
- 次回更新日が決まっている

## 8. 運用開始

月次運用:

- 表示内容確認
- 素材差し替え
- QRリンク確認
- 放映ログ確認
- 障害履歴確認
- 月次レポート提出
- 次月改善提案

社内記録:

- 契約条件
- 請求開始日
- 更新回数
- 対応時間
- 顧客要望
- 障害対応
- 追加提案

## 9. テスト導入から有償化

テスト導入終了時に出すもの:

- 表示写真
- 表示動画
- 放映ログ
- QR反応
- 担当者コメント
- 改善提案
- 有償プラン提案

有償化提案:

- Lite: 低初期で継続
- Standard: 月次更新とレポート込み
- Media: 広告枠運用込み

## 10. 失注・撤去

撤去時に行うこと:

- 顧客へ撤去日を連絡
- 端末・画面・金具を回収
- 端末ログと設定を退避
- 端末env/認証情報を削除
- Tailscale/VPN登録を削除
- 台帳ステータスを更新
- 失注理由を記録
