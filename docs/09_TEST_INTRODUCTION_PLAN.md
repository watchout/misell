# 09. Test Introduction Plan

## テスト導入の目的

テスト導入は、売上よりも事例化と運用検証を優先する。

目的：

- 3連表示の現場価値を確認する
- 設置場所ごとの視認性を確認する
- スタッフの反応を確認する
- QR/POPタグの反応を確認する
- 月額運用へ移行できるか検証する
- 営業資料に使える写真、動画、数値を取得する

## テスト導入条件案

### 無償/低額テスト

- 期間：2〜4週間
- 初期：実費または無償
- 月額：無償または低額
- 条件：写真撮影、事例化、フィードバック、レポート掲載許可

### 有償PoC

- 期間：1〜3か月
- 初期：98,000円〜298,000円
- 月額：49,800円〜79,800円
- 条件：月次レポート提出、継続判断

## テストで見るKPI

- QR読み取り数
- 表示素材別の再生回数
- スタッフ問い合わせ削減感
- 広告主候補の反応
- 来館者の目視反応
- 設置トラブル件数
- 素材更新工数
- 再起動/停止回数

## ヒアリング項目

店舗/施設側：

- どの場所に置くと見られるか
- 何を表示したいか
- 素材更新はどの頻度が理想か
- スタッフの手間は減りそうか
- 広告枠として使えると思うか
- 月額いくらなら継続できるか

広告主候補：

- 施設内広告に興味があるか
- QRクーポンを使いたいか
- 月額いくらなら出稿するか
- レポートで何が知りたいか

## テスト導入の成果物

- 設置写真
- 表示動画
- 管理画面スクリーンショット
- QRレポート
- 施設担当者コメント
- 改善点リスト
- 有償化提案書
- 次店舗展開資料

## テスト導入後の商談

1. テスト結果の報告
2. 効果があった表示パターンの説明
3. 月額運用プラン提示
4. 広告枠運用の提案
5. POPタグ/QR連動の提案
6. 複数店舗展開の提案

## 注意点

- テストで広告収益を保証しない
- AIカメラは最初から入れない
- 初期は人の目視とQRで十分
- 設置安全と配線を優先する
- 事例化の許諾を必ず取る
