# 60. テスト導入向け初回提案デック

## 目的

安心お宿、バリアン、パセラ、ビジョンセンターへ初回商談で提示できる、Misellの提案資料本文を定義する。

この資料はスライド化を前提にしている。各スライドの見出し、本文、話す要点、差し替え箇所をそのまま営業デックへ移せる状態にする。

## 提案の基本メッセージ

**連結サイネージ「ミセル -misell-」**

**つないで魅せる。施設の案内、販促、広告をひとつの店舗メディアへ。**

Misellは、3画面連結サイネージ、QR/POPタグ、月次レポートを組み合わせ、施設内の案内・販促・広告枠を運用できるサービスである。機材を売るのではなく、施設内の見える場所を継続運用できるメディアへ変える。

## 提案資料の使い方

- 初回商談では10から12枚で使う
- 価格は概要だけを入れ、詳細は `docs/61_CUSTOMER_PRICING_TABLE.md` へ分ける
- 広告枠の提案が強い相手には `docs/62_MISELL_MEDIA_KIT.md` を追加する
- 実機モニター到着前はブラウザデモ動画、管理画面スクリーンショット、WebM書き出し結果を使う
- 実機到着後は3画面表示写真と設置候補場所写真へ差し替える

## スライド構成

### 1. 表紙

**タイトル**

連結サイネージ「ミセル -misell-」ご提案

**サブコピー**

つないで魅せる。施設の案内、販促、広告をひとつの店舗メディアへ。

**入れる情報**

- 提案先施設名
- 提案日
- 提案者名

**話す要点**

「Misellは画面を置くだけのサイネージではありません。施設内で見られる場所を、案内・販促・広告に使える運用型メディアへ変える提案です。」

### 2. 施設内で起きている課題

**見出し**

紙POP、口頭案内、単発広告では、施設内の情報が届ききらない。

**本文**

- 館内案内やイベント情報が伝わらず、スタッフ説明が増える
- 紙POPやポスターの差し替えに手間がかかる
- 施設内サービスやキャンペーンが見落とされる
- 空き壁面、受付、導線が収益化されていない
- 広告や販促の反応が見えない

**話す要点**

「施設内には、すでに人が滞在していて、案内や販促を届けられる場所があります。ただ、紙や口頭だけでは更新性と計測性が弱いです。」

### 3. Misellの解決策

**見出し**

3画面連結表示で、案内・販促・広告を同時に届ける。

**本文**

- 左画面: 館内案内、補足情報、QR導線
- 中央画面: メイン広告、商品PR、イベント動画
- 右画面: クーポン、予約、POPタグ、広告主情報
- 3面wide: 目を引くキャンペーンや施設ブランド演出

**話す要点**

「3画面にすることで、単なる動画広告ではなく、案内、訴求、行動導線を同時に出せます。画面を増やす目的は派手にすることではなく、施設内の情報導線を整理することです。」

### 4. 3つの導入価値

**見出し**

案内が伝わる。販促が動く。広告収益へ広がる。

| 価値 | 施設側の効果 | 例 |
| --- | --- | --- |
| 案内 | スタッフ説明の負荷を減らす | 館内案内、混雑案内、イベント案内 |
| 販促 | 施設内サービスや商品を見せる | ルームサービス、フード、物販、回遊導線 |
| 広告 | 地域広告やスポンサー枠にする | 近隣店舗、求人、観光、イベント協賛 |

**話す要点**

「最初は案内や販促だけでも十分価値があります。運用が回り始めたら、広告枠やQR計測を追加して、施設のメディア価値を作れます。」

### 5. デモ表示イメージ

**見出し**

3画面で、施設ごとの使い方を切り替えられる。

**入れるビジュアル**

- 3面wideのデモ動画
- 左/中央/右のthree-zone表示
- LAN管理画面のPR作成フォーム
- 自然文から下書き反映の画面
- WebM書き出し結果

**話す要点**

「現時点のMVPでは、LAN管理画面から素材を登録し、商品PRを作り、再生成して置換し、WebMとして書き出せます。現場で素材差し替えと簡易PR制作ができることを重視しています。」

### 6. 施設別の活用パターン

| 提案先 | 最初に見せる価値 | 表示例 | 追加提案 |
| --- | --- | --- | --- |
| 安心お宿 | 館内案内と温浴/休憩訴求 | サウナ案内、混雑案内、朝食、近隣飲食 | 観光/周辺広告 |
| バリアン | 予約導線と客室サービス | 記念日、アメニティ、ルームサービス | 周辺飲食/再来店QR |
| パセラ | フード/イベント/推し活訴求 | コラボ、飲食、イベント、施設横断広告 | グループ内回遊 |
| ビジョンセンター | 会場案内とスポンサー枠 | 当日イベント、受付案内、配信パック | 企業広告/協賛枠 |

**話す要点**

「同じ仕組みでも、施設ごとに前面に出す価値は変えます。初回は施設がすぐ想像できる表示例を3つに絞って見せます。」

### 7. 運用フロー

**見出し**

導入後は、月次で表示を改善する。

1. 表示テーマを決める
2. 素材を登録する
3. 商品PRや広告素材を作る
4. スケジュール配信する
5. QR/放映ログを確認する
6. 月次レポートで改善する

**話す要点**

「設置して終わりではなく、月次で更新し、反応を見て改善します。これにより、画面が放置されるリスクを減らします。」

### 8. テスト導入プラン

**見出し**

まずは2から4週間のテスト導入で、現場価値を確認する。

| 項目 | 内容 |
| --- | --- |
| 期間 | 2から4週間 |
| 対象 | 1施設、1設置候補場所 |
| 表示内容 | 案内、販促、QR導線、広告枠サンプル |
| 成果物 | 表示動画、管理画面スクリーンショット、QR/放映ログ、改善メモ |
| 施設協力 | 設置場所確認、素材提供、フィードバック、事例化可否判断 |

**話す要点**

「テスト導入では広告収益を保証しません。まず、見られる場所か、施設側に運用価値があるか、QRや表示ログで反応が取れるかを確認します。」

### 9. 価格プラン概要

**見出し**

低初期で始め、月額運用と広告運用で広げる。

| プラン | 位置づけ | 月額目安 | 主な内容 |
| --- | --- | ---: | --- |
| Lite | 小規模/初回導入 | 49,800円から | 3連表示、月1回更新、簡易監視 |
| Standard | 基本プラン | 79,800円から | 月2回更新、QR計測、月次レポート |
| Media | 広告枠運用 | 128,000円から | 媒体資料、広告主対応、広告レポート |
| AI Edge | 上位拡張 | 個別見積 | AI制作、AIレポート、匿名カウント |

**話す要点**

「初期費用だけで売るのではなく、月額運用で継続的に価値を出します。詳細条件は価格表で提示します。」

### 10. テスト導入条件

**見出し**

テスト導入で確認することを先に合意する。

**確認項目**

- 設置候補場所で視認性があるか
- スタッフ説明負荷が減りそうか
- 施設内サービスや商品訴求に使えるか
- QRが読み取られるか
- 月額運用へ移行する価値があるか
- 事例写真/動画/コメントの掲載可否

**話す要点**

「テストの成功条件をあいまいにしないことが重要です。導入後に何を見て継続判断するかを、商談時点で決めます。」

### 11. 導入スケジュール

| フェーズ | 期間目安 | 実施内容 |
| --- | --- | --- |
| 現地確認 | 1日 | 設置場所、電源、LAN、視認性確認 |
| 素材準備 | 3から5営業日 | 初期表示素材、QR、広告枠サンプル |
| キッティング | 1から2営業日 | 端末設定、playlist、管理画面設定 |
| 設置/表示確認 | 1日 | 3画面表示、kiosk、自動起動確認 |
| テスト運用 | 2から4週間 | 表示、更新、ログ確認、改善 |
| レポート/継続提案 | 1週間 | 結果報告、本導入条件提示 |

**話す要点**

「実機到着後は3画面表示とkiosk起動を確認してから現場導入に進みます。現場で不確実な点は、テスト導入の確認事項として扱います。」

### 12. 次アクション

**見出し**

まず、設置候補場所とテスト導入条件を決める。

**依頼事項**

1. 設置候補場所の写真共有
2. 表示したい案内/販促テーマの選定
3. QR誘導先の候補確認
4. 広告枠利用可否の確認
5. テスト導入期間と担当者の確認

**話す要点**

「次に必要なのは、機材の詳細検討ではなく、どこに置くと見られるか、何を表示すると価値が出るかの確認です。」

## 施設別差し替えメモ

### 安心お宿

- 課題: 館内案内、温浴/休憩エリア訴求、近隣飲食送客
- 最初の表示: サウナ、休憩、朝食、混雑、周辺広告
- KPI: QR読み取り、スタッフ説明削減感、周辺広告候補数

### バリアン

- 課題: 客室サービス、記念日プラン、リピート予約導線
- 最初の表示: アメニティ、ルームサービス、記念日、予約QR
- KPI: QR読み取り、追加注文/予約導線反応、スタッフ評価

### パセラ

- 課題: フード/ドリンク、イベント、推し活/コラボ告知
- 最初の表示: 飲食訴求、イベント、施設横断広告、コラボ
- KPI: QR読み取り、注文導線反応、イベント告知反応

### ビジョンセンター

- 課題: 会場案内、当日イベント、スポンサー表示、配信パック訴求
- 最初の表示: 受付案内、当日スケジュール、スポンサー、配信サービス
- KPI: 案内負荷削減、スポンサー候補、問い合わせ導線反応
