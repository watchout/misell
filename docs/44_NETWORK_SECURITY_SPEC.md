# 44. ネットワーク・セキュリティ設計

## 目的

ミセル端末を店舗LANに接続して商用運用するため、ネットワーク、ファイアウォール、認証、アップロード制御、OS更新、リモート保守、データ削除の方針を定義する。

## 基本方針

- 店舗LANへ不要なポート開放を求めない
- 端末は外向き通信のみでクラウドと同期する
- 管理画面/adminはMVPではLAN内限定、商用では認証必須
- SSHは鍵認証のみ
- Tailscaleは管理用VPNとして扱う
- 素材アップロードは権限とファイル検証を必須にする
- OS自動更新は制御する

## 店舗LAN接続方針

### 推奨

- 有線LAN
- DHCP利用
- インターネット向き通信のみ許可
- 顧客の基幹ネットワークと分離できる場合は分離

### 望ましい構成

- 店舗ゲストLAN
- 専用VLAN
- サイネージ専用ルーター
- 顧客基幹システムと分離

## VLAN/ネットワーク分離

可能であれば、ミセル端末は以下のいずれかに置く。

1. サイネージ専用VLAN
2. ゲスト/IoT VLAN
3. 専用LTE/5Gルーター配下

避ける構成：

- POS/会計/予約システムと同じセグメントに無制限接続
- 管理画面を店舗LAN全体へ無認証公開

## 開放ポート方針

### 端末側 inbound

原則閉じる。

MVPでLAN内/adminを使う場合：

- 3000/tcp を同一LAN内に限定
- 商用時は認証またはVPN経由に限定

### 端末側 outbound

許可が必要な通信：

- HTTPS 443: クラウド同期
- DNS 53: 名前解決
- NTP 123: 時刻同期
- Tailscale利用時: Tailscaleが必要とする外向き通信

## ファイアウォール

Ubuntuではufwを使う。

初期方針：

- deny incoming
- allow outgoing
- allow ssh from Tailscale only
- allow admin port from LAN only during MVP

商用方針：

- adminはクラウド管理画面へ移行
- 端末ローカルadminは無効またはVPN限定

## /admin 認証方針

### MVP

- LAN内限定
- 簡易Basic認証または初期パスワード

### 商用

- 管理画面はクラウド側に集約
- ユーザー認証必須
- 役割権限を適用
- ローカルadminは無効化または保守用のみ

## 権限設計

### Admin

全店舗、全端末、全素材、全広告を操作可能。

### Operator

素材、playlist、レポート操作可能。

### Client Viewer

自店舗のプレビューとレポート閲覧のみ。

### Advertiser

自分の広告、素材、レポートのみ閲覧。

## 素材アップロードセキュリティ

必須：

- ファイルサイズ上限
- 拡張子チェック
- MIME typeチェック
- 保存ファイル名のランダム化
- 実行権限を付与しない
- path traversal対策
- サムネイル生成時の例外処理

許可拡張子：

- mp4
- webm
- jpg
- jpeg
- png

禁止：

- html直接アップロード
- js
- sh
- exe
- zip自動展開

HTML/URL表示は管理者のみ許可する。

## OS自動更新

### MVP

- 手動更新
- テスト後に適用

### 商用

- セキュリティ更新は定期メンテナンス枠で適用
- カーネル更新後は再起動テスト
- 自動更新で表示が止まらないよう制御
- unattended-upgradesは設定を慎重に行う

## アプリ更新

### 方針

- Git pull直更新はMVPのみ
- 商用ではバージョン管理されたリリースを配布
- rollback可能にする

### 更新手順

1. staging端末でテスト
2. 1台に先行適用
3. 問題なければ店舗群へ展開
4. エラー時は前バージョンへ戻す

## 秘密情報管理

端末内に保存する秘密情報：

- device_token
- API token
- Tailscale key

方針：

- Gitに含めない
- .envまたはconfig.jsonで管理
- 権限600
- 退役時に削除

## バックアップ

クラウド側：

- DB日次バックアップ
- 素材ストレージのバージョニング
- 月次エクスポート

端末側：

- config.jsonバックアップ
- playlist最新版
- assets manifest

## 監査ログ

管理画面で記録する操作：

- ログイン
- 素材アップロード
- playlist公開
- 広告公開
- 端末操作
- ユーザー権限変更
- 削除操作

## 端末紛失/盗難

対応：

1. device_token無効化
2. Tailscale登録削除
3. クラウドでlost状態に変更
4. 顧客へ通知
5. 端末交換
6. 必要なら警察/保険対応

端末内には個人情報を置かない設計にする。

## セキュリティチェックリスト

- SSH鍵認証
- rootログイン禁止
- ufw有効
- admin認証
- 素材アップロード検証
- device_token非公開
- ログローテーション
- 退役時削除手順
- OS更新方針
- 管理画面監査ログ

## 買収候補への見せ方

買い手は「端末が増えた時に事故らないか」を見る。

この設計により、ミセルは小規模MVPではなく、商用端末ネットワークとして拡張できることを示す。
