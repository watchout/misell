# 67. Misell Studio：NoviSignベンチマークCMS設計

## 目的

このドキュメントは、NoviSignで顧客がお金を払っている中核機能をベンチマークし、Misellの3連サイネージ特化CMSとして再設計するための仕様メモである。

重要: NoviSignをコピーするのではなく、サイネージ運用で実際に課金価値になっている機能を抽出し、Misellの強みである3連特化、施工込み、端末管理、将来AI拡張に寄せて実装する。

## 基本方針

Misell Studioは、汎用デジタルサイネージCMSではなく、3連サイネージ専用の運用CMSとして設計する。

NoviSignが証明している課金価値は以下である。

- 簡単に画面を作れる
- 遠隔でコンテンツを差し替えられる
- 曜日・時間帯で自動配信できる
- 端末が止まっていないか確認できる
- 実際に流れた証拠を確認できる
- 多拠点・複数画面を本部側で管理できる

Misellでは、これを3連サイネージに絞って実装する。

## 実装すべき中核機能

| 優先 | 機能 | 顧客がお金を払う理由 | Misellでの実装方針 |
| ---: | --- | --- | --- |
| 1 | レイアウトエディタ | 3連画面を自分たちで組める | 自由配置ではなく、3連テンプレート選択型から始める |
| 2 | プレイリスト/スケジュール | 時間帯別・曜日別に自動で流せる | 曜日、時間帯、期間、優先度、今すぐ公開を実装 |
| 3 | 素材管理 | 画像/動画をアップして使い回せる | サムネイル、タグ、使用中/未使用、容量表示を実装 |
| 4 | 端末管理/死活監視 | 止まっていないか分かる | heartbeat、最終通信、現在再生、エラー履歴を表示 |
| 5 | 再生ログ/Proof of Play | 広告・報告・改善に使える | Phase 1は簡易ログ、Phase 2でProof of Play強化 |

## 設計概念

NoviSign型のCMS概念を参考にしつつ、Misellでは以下の構造にする。

```text
Tenant
└── Site
    └── Display Wall
        ├── Screen 1 左
        ├── Screen 2 中央
        └── Screen 3 右

Asset
Layout
Template
Playlist
Schedule
Device
Playlog
Report
```

UI上の言葉は、できるだけ店舗担当者が理解しやすいものにする。

| 内部概念 | UI表示 |

## Misell Studioの主要メニュー

Phase 1では、管理画面のメニューを以下に絞る。

1. ダッシュボード
2. 素材
3. レイアウト
4. スケジュール
5. 端末管理
6. レポート

### 1. ダッシュボード

顧客と運用側が最初に見る画面。

表示するもの:

- 稼働中/停止中の端末数
- 今日の再生数
- エラー件数
- 最終通信
- 公開中のコンテンツ
- 次回スケジュール
- オフライン端末
- 容量警告

価値:

- 顧客に安心感を出す
- 運用担当が障害に気づける
- 月額費用の根拠になる

### 2. 素材

画像・動画を管理する画面。

Phase 1で実装するもの:

- 画像アップロード
- 動画アップロード
- サムネイル表示
- ファイル名編集
- タグ管理
- 使用中/未使用表示
- 容量表示
- 削除

Phase 2以降:

- Google Driveインポート
- Dropbox/SharePoint連携
- CSVインポート
- 素材承認フロー
- ブランドキット

### 3. レイアウト

Misell Studioの最重要画面。

方針:

- 最初は完全自由配置エディタにしない。
- 3連サイネージで実用性が高いテンプレート選択型にする。
- 顧客が迷わないUIを優先する。
- 3連プレビューを常に表示する。

Phase 1テンプレート候補:

- 3画面それぞれ別素材
- 3面ぶち抜き動画
- 左: メニュー / 中央: 動画 / 右: QR
- 左右: 静止画 / 中央: 動画
- 中央メイン + 左右広告
- 左メイン + 中央/右サブ
- 右QR固定
- 上下テロップ付き
- ホテル館内案内
- 飲食メニューボード
- イベント案内
- キャンペーン告知

Phase 2以降:

- Videowall Editor相当の自由度強化
- Zone単位の比率変更
- テロップウィジェット
- QRウィジェット
- RSS/天気/カレンダーウィジェット
- 外部データ埋め込み

### 4. スケジュール

顧客が課金価値を感じる重要機能。

Phase 1で実装するもの:

- 曜日指定
- 開始時刻
- 終了時刻
- 掲載開始日
- 掲載終了日
- 優先度
- 通常配信
- 臨時配信
- 今すぐ公開
- プレビュー

ユースケース:

- 朝/昼/夜メニューの切り替え
- 週末だけキャンペーン表示
- イベント当日だけ案内表示
- ホテルの時間帯別案内
- 複数素材のローテーション

### 5. 端末管理

NoviSignのDevice Manager相当。Misellでは施工・保守込みのため、特に重要。

Phase 1で実装するもの:

- 端末名

- 最終通信

- アプリバージョン

- 同期状態
- エラー履歴
- 再起動要求

社内運用向けに持つ情報:

- 設置場所
- モニター型番

- 施工日
- 保守期限
- 施工写真

### 6. レポート

Phase 1は簡易レポートでよい。

Phase 1:

- 日別再生数
- 素材別再生数
- 端末稼働率
- エラー件数

## UI/UX方針

NoviSignに寄せるべき点:

- 非エンジニアでも使える
- ドラッグ&ドロップ的な分かりやすさ
- テンプレートから始められる
- 素材、レイアウト、スケジュール、公開の流れが自然
- 端末状態が見える
- レポートで運用成果を確認できる

Misellで変えるべき点:

- 汎用CMSではなく3連に特化する
- 自由度より迷わないことを優先する
- 完全自由配置ではなく制約付きテンプレートから始める
- 施工・保守・端末管理の情報をCMSに統合する
- 3画面横並びプレビューを中心に置く

推奨操作フロー:

1. テンプレートを選ぶ
2. 左/中央/右の枠に素材を入れる
3. 再生順を決める
4. 曜日と時間を決める
5. プレビューする
6. 公開する
7. 端末状態と再生ログを確認する

## 3連専用プレビュー

Misell独自価値として、3連専用プレビューを必須にする。

必要機能:

- 左・中央・右を横並びで表示
- 3面wide動画のプレビュー
- 各画面の素材切れ警告
- 画面比率ミス警告
- 文字が小さすぎる場合の警告
- QRサイズ警告
- 端末解像度との差分警告

表示モード:

- 実寸比率プレビュー
- 簡易プレビュー
- 1画面拡大プレビュー
- 全体wideプレビュー

## Codex向け実装指示メモ

Misell StudioはNoviSign型のCMS概念を参考にする。ただし、汎用CMSではなく3連サイネージ特化で実装する。

Phase 1では自由配置エディタではなく、テンプレート選択型レイアウトエディタにする。顧客がお金を払う中核機能は、素材管理、3連レイアウト、プレイリスト、スケジュール、端末監視、再生ログである。

Business Plus相当のProof of Play、Videowall Editor強化、外部連携、複数店舗管理はPhase 2に送る。Premium相当のSSO、権限、承認フロー、監査ログはPhase 3に送る。
